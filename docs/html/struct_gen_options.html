Detailed Description

A type that contains choices of settings for SimData functions that create a new AlleleMatrix/generation.

The family_size parameter will affect how many offspring are produced.

The will_name_subjects, will_track_pedigree, and will_allocate_ids parameters affect how much extra detail about the offspring is generated/saved.

The will_save_to_simdata toggle allows you the option of generating offspring without saving them in memory. This may be useful in combination with save-as-you-go toggles will_save_pedigree_to_file, will_save_effects_to_file, and will_save_genes_to_file, to generate a larger number of offspring than will fit in memory.

Parameters
will_name_subjectsa boolean representing if subject_names will be filled or not.
subject_prefixIf will_name_subjects is true, subjects are named [subject_prefix][index].
family_sizethe number of offspring to produce from each cross.
will_track_pedigreea boolean representing whether to bother to track parentage of individual outcomes of the cross or not.
will_allocate_idsa boolean representing whether to allocate individuals session-unique ids used for pedigree tracking.
filename_prefixa string
will_save_pedigree_to_filea boolean. If true, the full/recursive pedigrees of every genotype generated in the cross are saved to "[filename_prefix]-pedigree", even if the genotypes are not later saved to SimData.
will_save_effects_to_filea boolean. If true, the breeding values of every genotype generated in the cross are saved to "[filename_prefix]-eff", even if the genotypes are not later saved to SimData.
will_save_genes_to_filea boolean. If true, the set of alleles of every genotype generated in the cross are saved to "[filename_prefix]-genome", even if the genotypes are not later saved to SimData.
will_save_to_simdataa boolean. If true, the offspring are retained in the SimData as a new group. If false, they are discarded after creation.
